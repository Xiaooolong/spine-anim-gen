## 优化与新功能实施方案

为了提升用户体验并满足“快速测试本地文件”的需求，我制定了以下计划：

### 1. UI 表现优化 (不改动原有逻辑)
- **布局升级**：引入一个通用的 `Layout` 组件，添加美观的侧边栏导航，使页面切换更加顺畅。
- **视觉美化**：
  - 使用 Tailwind CSS 重新设计按钮、卡片、输入框和列表。
  - 引入 `lucide-react` 图标库，为每个操作（上传、列表、预览、返回）增加直观的图标。
  - 优化任务列表的展示，增加状态颜色标识和进度条。
  - 改进任务详情页的响应式布局，使视频和 Spine 预览并排展示更合理。

### 2. 实现“快速测试/预览”功能
- **新建快速预览页**：在侧边栏增加“快速预览”入口，允许用户直接上传或拖拽本地的 Spine 文件。
- **支持本地文件拖拽**：
  - 用户可以一次性选择 `.json`、`.atlas` 和 `.png` 文件。
  - 使用 `URL.createObjectURL` 在浏览器内存中创建临时链接。
- **增强预览组件**：
  - 升级 `SpinePixiPreview` 组件，使其能够智能识别并加载本地 Blob 资源。
  - 通过 `PIXI.Assets.add` 手动注册资源映射，解决本地 JSON 引用 Atlas/PNG 的路径问题。
- **纯前端实现**：此功能将完全在前端浏览器中完成，无需经过后端 API 处理，满足快速查看效果的需求。
